# Definition of Done (DOD)

## Overview

This document defines the criteria that must be met for a version of AniMathIO to be considered "Done" and ready for release.

## Acceptance Criteria

A version is considered **Done** when all of the following criteria are satisfied:

### 1. Testing & Integration
- [ ] All features and bug fixes have been thoroughly tested in the `staging` branch
- [ ] All tests pass successfully
- [ ] Code has been reviewed and approved
- [ ] Changes have been merged into the `main` branch

### 2. Documentation
- [ ] Comprehensive documentation has been written and published to [docs.animathio.com](https://docs.animathio.com)
- [ ] Documentation includes:
  - Installation instructions
  - Usage examples
  - API references (if applicable)
  - Changelog/release notes
- [ ] Documentation updates are committed to the [AniMathIODocs repository](https://github.com/AniMathIO/AniMathIODocs)

### 3. Distribution
- [ ] The new version is available for download from [animathio.com](https://animathio.com)
- [ ] Download links are functional and verified
- [ ] Version information is correctly displayed on the website
- [ ] Website updates are committed to the [AniMathIOWebsite repository](https://github.com/AniMathIO/AniMathIOWebsite)

## Related Repositories

- **Main Project**: [AniMathIO/AniMathIO](https://github.com/AniMathIO/AniMathIO)
- **Documentation**: [AniMathIO/AniMathIODocs](https://github.com/AniMathIO/AniMathIODocs)
- **Website**: [AniMathIO/AniMathIOWebsite](https://github.com/AniMathIO/AniMathIOWebsite)

## Notes

- All three criteria must be completed before a version can be considered done
- Any deviation from these criteria must be documented and justified
- This DOD should be reviewed and updated as the project evolves
